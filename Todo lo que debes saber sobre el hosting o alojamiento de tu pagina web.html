Alojamiento web o hosting


Publica tu sitio web

Ten encuenta varios aspectos a la hora de escoger un nombre de dominio

    Nombre

  • El nombre de tu página web debe ser preciso y explicativo, eso te permitirá tener un mejor posicionamiento en los buscadores y ser mucho más llamativo para los usuarios.
  • Existen nombres de dominios no disponibles, como por ejemplo Google.com o YouTube.com. Sé creativo, busca nombres llamativos.

    • Extensión

  • Entre más reconocida sea la extensión (.com .net ) el precio de adquisición por el dominio tiende a aumentar.

    • Tiempo

  • El dominio no se vende por tiempo ilimitado, lo normal es que consigas planes anuales o mensuales. Así que ten cuidado y ten presente la fecha en la que debes actualizar tu dominio para que nadie más se te adelante.

Para tener en cuenta en tu alojamiento o servidor web

    Servidor

  • Como servidor web existen Servicios para almacenar informacion Web ya sean, paginas, correos, etc.

    • Alojamiento

  • Como alojamiento para tu sitio web existen alternativas gratuitas que son recomendables para practicar o probar diseños. Por otra parte, si se quiere un alojamiento web para un sitio profesional, lo más aconsejable es pagar por uno adecuado, para así evitar publicidad fastidiosa y tener mejores herramientas.

Opciones para servidores o alojamientos web pagos

  • Existen dos alternativas entre los servidores que se explican en este apartado, se diferencian fundamentalmente en el lugar donde almacenan la información. El primer tipo de servidor es el servidor que almacena su información en una instalación física, como un monitor o varias torres de computador. El segundo tipo almacena su información en la nube, en donde los servidores son completamente virtuales, son más seguros y más flexibles, pues tiene más prestaciones.

    • Compartido:

  • Es el más común, debido a su relación calidad-precio. Adicionalmente, el servicio se encarga de la mantención del servidor, por lo que no tendrás que preocuparte por eso. Por otro lado, compartes los recursos con otros alojamientos independientes al tuyo. Normalmente se utiliza para sitios web que tienen un tráfico pequeño. Para mas informacion Wikipedia

    • Servidor virtual privado (VSP):

  • una opción más segura y estable que el servidor compartido, porque se tiene más control del sitio web a un precio mayor. Normalmente se usa para sitios web que tienen un tráfico medio. Para mas información Hostinger Tutoriales

    • Servidor dedicado:

  • Mejores prestaciones en el alojamiento y control completo del servidor; la diferencia fundamental que tiene con el VSP es que la información se guarda en espacios físicos, en un ordenador que puede ser comprado o alquilado. Es decir, necesitas un gran conocimiento en la manutención de los servidores, ya que estos correrán por tu cuenta. ¿VPS o Dedicado?



Criterios para comprar HOST

Calidad

  • ¿Qué tanta seguridad te da el sitio web, teniendo en cuenta que cumpla con todo lo que te prometieron antes de la contratación?
  • Deberá resolver efectivamente los problemas que tengamos
  • Tiene que tener buena disponibilidad (que no esté apagado frecuentemente)

Indicadores para medir la calidad del servicio de Alojamiento o Hosting

  • Calidad web de la empresa: el aspecto que tenga la página web hablará mucho sobre la empresa
  • Opiniones de los clientes
  • Tiempo operando: la experiencia es fundamental para estos escenarios, aunque pueda sonar prejuicioso; tengamos en cuenta que a mayor tiempo operando la experiencia será mayor. Adicionalmente, si ha podido prosperar en el mercado x tiempo, es porque tiene algo bueno que ofrecer a sus clientes.
  • Soporte al cliente: el tiempo de respuesta que debe tener la empresa contratada debe ser mínimo frente a cualquier problemática que se pueda llegar a presentar.
  • Buscar comparativas recientes que nos ayuden a tomar una mejor decisión

Sistema Operativo

  • Es importante tener en cuenta que según el sistema operativo en el que estés trabajando vas a necesitar un servicio diferente. En el caso de Linux el alojamiento web tiende a ser más económico que el host en Windows.

Almacenamiento

    ¿Qué puede llegar a incluir?

  • Puede incluir bases de datos o plantillas que abarquen todo el espacio gratuito o pago disponible obligándonos a pagar más, los que tengan esta característica no tienen mucha seriedad como empresa

    • ¿Qué almacenamiento necesito?

  • Se realiza un cálculo de aproximación, el cual se aumentará un 20%
  • El cálculo de aproximación se saca multiplicando el número de páginas que tenga el sitio web por el tamaño medio de la pagina
  • Para sacar el tamaño medio de la página es necesario hacer una suma de todos los archivos que tenga la página web, CSS, HTML, JavaScript e imágenes

Transferencia

  • La unidad por la cual se mide son gigabytes/mes la cual calcula la capacidad del volumen de transferencia que tiene el host
  • El volumen de transferencia es difícil de calcular ya que mide el número de visitas que tendrá la página web.
  • Aunque algunas empresas usan el slogan de "Transferencia ilimitada" no es correcto debido a que se necesita un lugar físico para almacenar dichas trasferencias

    • ¿Qué volumen de transferencia necesito? y ¿Cómo calcularlo? aproximadamente

  • Multiplicar número de visitantes X paginas X usuario
  • El resultado multiplicarlo X el tamaño medio de la pagina
  • El resultado multiplicarlo X 30 (dias del mes)

    Velocidad de transferencia

  • Ancho de banda
  • No existe un ancho de banda ilimitado
  • Unidad: Kbps - Mbps
  • Entre más ancho de banda el tiempo de espera por parte del usuario será menor
  • El ancho de banda es compartido por el número de visitantes que haya al mismo tiempo en la pagina

Lenguaje de programacion y Base de datos

  • ¿Qué lenguaje de programación admite el servicio? ¿Me es útil ese lenguaje?
  • Para las bases de datos hay que tener en cuenta que se necesitan diversos almacenamientos según el sistema operativo que se necesite

Dominios

    Subdominios

  • Divisores del dominio, por ejemplo carros/azules.com, en este caso /azules sería un subdominio del dominio carros.com

    • Dominios

  • Nombre para el sitio web
  • Se pueden usar varios dominios para una sola página web como estrategia de marketing

Otros

  • Cuentas de correo electrónico, FTP y SSH
  • Scripts y librerías preinstaladas
  • Aplicaciones preinstaladas, Foros, blog, gestor de contenido, comercio electrónico
  • Estadísticas de visitas y ficheros de registros (Logts-> Donde se almacenan datos estadísticos)
  • Copias de seguridad
  • Procesos en segundo plano que se activen de forma periodica (CRON)

  • Calidad de los paquetes completos cual es mejor entre X página o Y página
